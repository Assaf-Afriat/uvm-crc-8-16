1. DUT Specification

Requirements & interface — configurable CRC-8/16, serial or parallel

Summary

The DUT is a configurable CRC block supporting CRC-8 or CRC-16 with a configurable polynomial. It operates in serial mode (1 bit per cycle) or parallel mode (e.g. 8 bits per cycle). A reference model in the scoreboard computes the expected CRC; verification compares DUT output to the reference. Coverage includes multiple polynomials and serial vs parallel modes.

Requirements (R1–R7)

IDRequirement
R1Serial mode produces the same CRC as the reference for any input bit sequence.
R2Parallel mode produces the same CRC as the reference for any input byte/word sequence.
R3CRC width is 8 or 16 bits (parameter or mode).
R4Polynomial is configurable; at least two presets (e.g. CRC-8, CRC-16-CCITT).
R5Reset clears internal state; next computation uses configured init value.
R6Init value is configurable (e.g. 0 or 0xFFFF).
R7Optional final XOR (e.g. 0 or 0xFF/0xFFFF) configurable for standard compliance.

Interface (data path)

SignalDirWidthDescription
mode_serialin11 = serial (1 bit/cycle), 0 = parallel (e.g. 8 bits/cycle).
data_inin1 or 8Input data; meaning per mode.
data_validin1Input valid; DUT consumes data this cycle.
startin1Start new CRC (reload init, clear state).
crc_outout8 or 16CRC result.
crc_validout1CRC result valid 1 cycle after last data_valid.

2. Verification Plan

UVM testbench architecture, test list, coverage plan

Objectives & sign-off criteria

Objectives: Prove correct CRC for any valid input (serial/parallel) and any configured polynomial, init, and final XOR; prove reset and start re-initialize state; prove CRC-8 and CRC-16 per R1–R7.

Sign-off: All R1–R7 have at least one passing test and documented coverage path; functional coverage on polynomial presets (0x07, 0x1021), serial/parallel, and data patterns; no open P1/P2 bugs; regression passes.

Testbench architecture

Top: tb_top.sv, tb_top_init_final_xor.sv, tb_top_crc16.sv — clock, reset, DUT, config_db, run_test().

Agent: CrcAgent (CrcDriver, CrcMonitor, CrcSequencer). TLM: Monitor analysis_port → Scoreboard + Coverage. Reference: CrcScoreboard.ref_crc() — Galois LFSR, LSB first, init/final XOR. Config: CrcConfig (width, polynomial, init, final_xor) via uvm_config_db.
